Aide

Mode d'emploi :
1. Cliquez sur un nœud (ou une question de la liste à gauche, ou une réponse de son menu déplié) pour le modifier dans le panneau de droite.
2. Une fois vos modifications faites dans le panneau de droite, cliquez sur « Valider » pour les appliquer à l'arbre.
3. La forme de l'arbre est calculée automatiquement : vous ne pouvez pas déplacer un nœud à la main, elle se réorganise toute seule dès que vous ajoutez/supprimez une question, une recommandation ou une branche.
4. Si un nœud est sélectionné (en surbrillance), cliquer sur « + Question », « + Étape » ou « + Recommandation » crée directement le lien avec ce nœud. Sinon, utilisez « Relier deux nœuds » : cliquez le parent puis l'enfant.
4bis. Question vs Étape : une Question a toujours des réponses nommées. Une Étape sert pour un passage intermédiaire qui ne pose pas vraiment de question : laissez le nom de la branche vide si elle n'a qu'une seule suite, elle s'exportera comme un simple enchaînement (sans « Branche 1 », « Branche 2 » inventés dans le JSON).
5. Pour vous déplacer : cliquez-glissez sur le fond, utilisez les barres de défilement, ou la molette de la souris au-dessus de l'arbre pour zoomer/dézoomer.
6. Utilisez la barre de recherche pour retrouver un nœud par mot-clé : l'arbre se centre sur lui et le sélectionne pour modification immédiate.
7. Une même réponse peut mener à plusieurs décisions/questions à la fois : sélectionnez la réponse (la branche) puis cliquez sur « + Question » ou « + Recommandation » pour en ajouter une autre dessus.
8. Le bouton « ⛶ Plein écran arbre » masque la liste des questions à gauche pour agrandir la vue de l'arbre. Le ✕ en haut du panneau de droite permet de le fermer ; il se rouvre automatiquement à la prochaine sélection.
9. Ctrl+Z pour annuler, Ctrl+Maj+Z pour rétablir.

Mise à jour GitHub :
1. Cliquez sur « Vers GitHub ».
2. Allez dans le dépôt GitHub de l'Atlas.
3. Supprimez l'ancien arbre_dynamique.json.
4. Ajoutez le fichier téléchargé.
5. Vérifiez qu'il s'appelle exactement arbre_dynamique.json.
6. Cliquez sur « Commit changes ».
